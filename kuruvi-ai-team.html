I Built an AI Team That Manages Itself (And Cut My Costs by 80%)

How switching to the "expensive" model actually saved me money

Aravindh • February 2026 • 8 min read

Last month, my AI bill was crushing me. Not because I was using ChatGPT for everything — I had built my own agent system called Nanoclaw. But here's the thing: having AI agents is easy. Having them not step on each other's toes? That's the hard part.

So I did something counterintuitive. I upgraded every single agent to Opus (the most expensive Claude model) and added a whole orchestration layer on top. My costs dropped 80%.

Here's how.

The Problem: AI Anarchy

I built Nanoclaw to be my personal assistant team. I had agents for coding, design, testing, architecture, product strategy, and research. The interface was WhatsApp. I'd message, agents would respond. Worked great... until it didn't.

The breaking point: I asked Turing to implement a feature. Nitty started testing before Turing finished. Jony started designing UI for code that didn't exist yet. Everything collided. I spent 3 hours untangling the mess.

The real issue? No orchestration. Agents were just workers with no manager.

Opus costs 3x more per token. But when it actually completes the task the first time, you use way fewer tokens overall.

Enter Kuruvi: The Orchestration Layer

I built Kuruvi to solve the coordination problem. Here's how it works:

1. Task Queue System

Instead of agents running simultaneously, Kuruvi maintains a task queue. Each agent only starts when the previous one finishes. No collisions.

2. Docker Isolation

Every agent runs in its own Docker container with a fresh Claude session. When the task is done, the container dies. No context pollution.

3. Convex Backend

All task state lives in Convex (real-time database). Agents can see what's in progress, what's queued, what's done. Full visibility, zero race conditions.

4. Discord Notifications

Every completed task posts to Discord. I get a notification, can review the work, and approve or request changes.

The magic: I went from managing 6 chaotic AI agents to managing 1 orchestrator that manages 6 disciplined AI agents. The cognitive load dropped by 90%.

Why This Matters

Everyone's building AI agents. But here's the thing: the hard part isn't the agent. It's the orchestration.

When you have one AI, you manage it. When you have ten, they need to manage each other. That's the unlock.

Lessons Learned

  • Quality > Quantity: One Opus call that works beats three Sonnet retries
  • Orchestration is Everything: Context matters more than capability
  • Fresh Sessions > Long Context: Kill the container, start clean
  • Async > Real-time: Task queues let agents work without blocking

The Bottom Line

I replaced 20 hours of manual AI wrangling per week with a $35/month orchestration system. The future isn't one super-intelligent AI. It's a bunch of specialized AIs that know when to talk and when to shut up.
